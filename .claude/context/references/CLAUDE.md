# References Directory

This directory contains detailed reference documentation that can be loaded on-demand.

## Purpose

Progressive disclosure pattern: Keep main context files concise (<500 lines) and store detailed documentation here for when it's needed.

## Usage

When you need detailed information on a topic, read the relevant file from this directory.

## Contents

Add reference files here as needed:
- `api-reference.md` - API documentation
- `architecture-details.md` - Detailed system architecture
- `troubleshooting.md` - Common issues and solutions
- `conventions.md` - Coding conventions and patterns
